## 오늘한일

### TIL 블로그 만들기

1. gatsby sample project clone
2. github repo 생성
3. 생성된 repo에, main,gh-pages 이름으로 2개 branch 생성.

```
github repo 위치에서,
git remote add gatsbyProject path/to/gatsbyProject
git fetch gatsbyProject
git merge --allow-unrelated-histories sample/branch-name
git remote remove projectA

```

#### gitFork 에서, 안됨. 콘솔 커맨드로 해결.<br/>git command 학습이 필요.

4. gh-pages 적용

```
yarn add --dev gh-pages

```

```js
//gatsby-config.js
module.exports = {
  pathPrefix: "/reponame",
}
```

```json
//package.json
{
  "scripts": {
    "deploy": "gatsby build --prefix-paths && gh-pages -d public"
  }
}
```

이렇게 설정하고, yarn deploy 엔터 하면,

현재 브랜치에서 빌드완료한 public 폴더가, gh-pages 브랜치로 푸시됨
